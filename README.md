# www

A fake 1990s internet server. Point a retro computer at it and browse a procedurally-generated Geocities-era web — complete with garish colour schemes, blinking text, marquee headers, and a Yahoo!-style search engine called **Boohoo!**

Pages and search results are generated on the fly by a local LLM running via [llama.cpp](https://github.com/ggerganov/llama.cpp). No internet connection required after setup.

---

## What it does

- **Boohoo! home page** — Yahoo! 1996-style directory with category links and a search form
- **Search** — generates 6 fake results for any query, each linking to a content page
- **Content pages** — full Geocities-era HTML 3.2 pages generated by one of 11 rotating personas; deliberately inaccurate "facts" are highlighted in red italic
- **Boohoo! Chat** — a Yahoo! Chat-style room staffed by `h4x0r_1337`, a leet hacker who actually answers technical questions

All output is plain HTTP, no JavaScript, no CSS — compatible with Mosaic, Netscape 3, Lynx, and anything else that speaks HTTP/1.0.

---

## Prerequisites

- CMake 3.14+
- A C++17 compiler (GCC 8+ or Clang 7+)
- git (for FetchContent to clone dependencies)
- wget or curl (for the model download script)
- 2–8 GB RAM depending on which model you run

CMake's FetchContent pulls llama.cpp and cpp-httplib automatically on the first build — no manual submodule steps needed.

---

## Build

```bash
git clone <this-repo>
cd www
mkdir -p build && cd build
cmake ..
make -j$(nproc)
```

The first build clones llama.cpp and compiles it, which takes a few minutes. Subsequent builds are fast.

The binary lands at `build/www`.

---

## Get a model

```bash
./scripts/download_model.sh
```

Interactive menu with five options:

| Model | Size | Notes |
|---|---|---|
| Llama 3.1 8B Instruct | 4.9 GB | Recommended — best output quality |
| Mistral 7B Instruct v0.3 | 4.4 GB | Fast, good results |
| Qwen 2.5 7B Instruct | 4.7 GB | Strong instruction following |
| Llama 3.2 3B Instruct | 2.0 GB | Lighter — works on 4 GB RAM |
| Qwen 2.5 1.5B Instruct | 1.0 GB | Pi-friendly — runs on 2 GB RAM |

All are Q4_K_M quantizations from [bartowski's HuggingFace repos](https://huggingface.co/bartowski). The script also accepts a custom HuggingFace URL.

Models are saved to `models/`.

---

## Run

```bash
./build/www --model models/Qwen2.5-1.5B-Instruct-Q4_K_M.gguf --port 8080
```

Then open `http://<your-machine-ip>:8080` in any browser, including retro ones.

### Options

| Flag | Default | Description |
|---|---|---|
| `--model <path>` | *(required)* | Path to a `.gguf` model file |
| `--port <n>` | 80 | HTTP port to listen on |
| `--threads <n>` | 4 | CPU threads for inference |
| `--ctx <n>` | 4096 | KV cache size in tokens |
| `--gpu-layers <n>` | 0 | Transformer layers to offload to GPU |

### Tips for retro hardware

- Requests block while the LLM generates — old browsers will sit on the loading screen for 30–120 seconds depending on model and hardware. This is normal and intentional.
- Read/write timeouts are set to 5 minutes to accommodate slow generation.
- Only one request can run at a time (single inference context). Concurrent requests queue behind the mutex.
- On a Pi or other low-RAM machine, use the 1.5B model and set `--threads` to match your core count.

## Routes

| Route | Description |
|---|---|
| `GET /` | Boohoo! home page |
| `GET /search?q=<query>` | Generate 6 fake search results |
| `GET /page?topic=<title>&q=<query>` | Generate a full Geocities content page |
| `GET /chat?msg=<message>` | Single-turn chat with `h4x0r_1337` |

## How it works

1. **Request arrives** — the httplib route handler extracts query parameters
2. **Prompt construction** — `main.cpp` builds a system prompt (persona + style rules) and a user prompt (topic/query)
3. **Inference** — `Generator::generate()` applies the model's chat template, tokenises, runs the prefill pass, then samples tokens autoregressively until EOS or the token budget is exhausted
4. **Post-processing** — `renderer.cpp` strips any model preamble, encodes code inside `<PRE>` blocks, fixes unreadable colour contrast, converts markdown fences to HTML, and highlights false facts
5. **Response** — the assembled HTML is sent directly to the client

Token budgets: 600 for search results, 1500 for content pages and chat.
